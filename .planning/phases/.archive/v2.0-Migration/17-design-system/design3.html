Library: shadcn/ui vs. Scratch
Building from scratch offers total control, but shadcn/ui provides the "copy-paste ownership" model. It uses Radix UI primitives for accessibility (ARIA, focus management) while letting us inject our Soft Neobrutalist CSS directly into the component files.
REC: shadcn/ui (Ownership + Speed)
Motion: Framer vs. CSS
"Lots of good animations" requires Framer Motion.
  • LAYOUT layoutId for seamless shared element transitions.
  • GESTURES Built-in drag/hover/tap physics.
  • LOTTIE Use for complex "Success" celebrations.
Professional tools like Linear use Framer Motion for that "snappy" reactive feel.
REC: Framer Motion (Snappy/Reactive)
Validation: UI vs. Logic
Logic (Zod) determines *if* a date is valid (e.g., ETA filed < 180 days after Recruitment). The UI is how we communicate it.
const CaseSchema = z.object({

});
Connection: Use React Hook Form's errors object to trigger "Hard Shadow" red alerts and micro-shake animations.
REC: Zod + React Hook Form + Integrated UI Hooks
Skeleton Strategy
These are Design System concerns. We define <Skeleton /> primitives (Block, Circle, Text) that follow our monolithic 2px border rules.
Pages then compose these into "Loading States" that match the actual component layout perfectly.
REC: Design System Primitives
Visual Identity: Heavy-Ink Monolithic Blockwork
